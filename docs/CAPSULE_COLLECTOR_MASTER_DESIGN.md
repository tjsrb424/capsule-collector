# 캡슐 컬렉터 MASTER DESIGN

> 문서 목적  
> 이 문서는 `캡슐 컬렉터` 신규 프로젝트의 장기 방향, 시스템 구조, UI/UX 원칙, 커피 테마 1차 구현 범위, 확장 규칙, Cursor Auto 작업 기준을 정리한 마스터 기획 문서다.  
> 이 게임은 단일 테마 게임이 아니라, 여러 게임/세계관의 카드를 테마 단위로 계속 추가하는 **수집형 캡슐 오픈 게임 플랫폼**이다.

---

## 0. 핵심 정의

### 게임명

**캡슐 컬렉터**

### 장르

- 수집형 카드 컬렉션
- 캡슐 오픈/뽑기형 캐주얼 게임
- 광고 보상형 캐주얼 게임
- 테마 확장형 컬렉션 허브

### 첫 번째 구현 테마

**커피 테마**


## 1. 제품 방향성

### 1.1 한 줄 기획

**여러 게임 세계관의 카드를 캡슐로 수집하고, 도감과 카드 성장, 테마별 완성 보상을 통해 계속 모으고 싶게 만드는 감성적인 수집형 게임.**

### 1.2 핵심 감정

이 게임은 복잡한 조작보다 다음 감정을 강하게 전달해야 한다.

- 캡슐을 열기 직전의 기대감
- 새 카드가 나왔을 때의 만족감
- 희귀 카드가 떴을 때의 짜릿함
- 도감 빈칸을 채워가는 수집 욕구
- 예쁜 카드를 바라보는 감성적 만족
- 중복이 나와도 성장에 쓰이는 안정감
- 오늘도 한 번 더 열고 싶은 가벼운 루틴감

### 1.3 최우선 UX 키워드

- 깔끔함
- 모던함
- 감성적
- 부드러운 애니메이션
- 프리미엄 캐주얼
- 카드 수집 욕구
- 도감 완성 욕구
- 짧고 명확한 보상 루프
- 모바일 세로 화면 최적화

### 1.4 피해야 할 방향

- 너무 복잡한 전투/전략 시스템
- 광고만 강요하는 구조
- 보상이 과하게 난잡한 구조
- 카드가 많지만 갖고 싶지 않은 구조
- 도감이 단순 리스트처럼 보이는 구조
- 애니메이션이 과하거나 버벅이는 구조
- 처음부터 서버/로그인/결제/광고 SDK까지 붙이는 구조

## 2. 핵심 게임 루프

### 2.1 기본 루프

```text
1. 캡슐 획득
2. 캡슐 열기
3. 카드 획득
4. 신규/중복 판정
5. 신규 카드는 도감 등록
6. 중복 카드는 성장 재화로 변환
7. 카드/테마 수집률 상승
8. 수집 점수 증가
9. 더 좋은 캡슐 또는 특정 테마 캡슐에 도전
```

### 2.2 감정 루프

```text
기대감 → 공개 연출 → 획득 만족 → 도감 채움 → 성장 욕구 → 다시 캡슐 오픈
```

### 2.3 광고 보상 루프

광고는 강제형보다 선택형으로 설계한다.

```text
캡슐이 부족하다
→ 광고 보고 보너스 캡슐 열기

중복 카드가 나왔다
→ 광고 보고 중복 보상 2배

일일 보상을 받았다
→ 광고 보고 보상 2배

시간대 한정 캡슐이 열렸다
→ 광고 보고 추가 1회 열기
```

- 모든 테마는 `themeId` 기반으로 확장한다.

- 캡슐 설정도 테마별로 분리한다.
- 유저 보유 상태는 카드 원본 데이터와 분리한다.
- 뽑기/점수/중복/진행률 계산은 UI와 분리된 순수 함수로 만든다.
- 초기 저장은 로컬스토리지 기반으로 한다.
- 서버 저장은 v1 이후 별도 단계에서 붙인다.

## 6. 커피 테마 기획

### 6.1 커피 테마 개요

커피 테마는 Coffee 2048의 음료 카드 80종을 수집하는 첫 번째 테마다.

### 6.2 카드 수

총 80장.

### 6.3 카테고리 구조

총 10개 카테고리, 각 8장.

```text
1. 에스프레소/기본 커피
2. 기본 우유 커피
3. 시럽/달달 라떼
4. 모카/디저트 커피
5. 말차/티 라떼
6. 청량/아이스/논커피
7. 희귀 재료 음료
8. 시간대 한정 음료
9. 시그니처 음료
10. 전설/고급 컬렉션 음료
```

### 6.4 등급 구조

```text
common
uncommon
rare
signature
legendary
```

### 6.5 커피 테마의 수집 목표

- 모든 커피 카드 80장 수집
- 각 카테고리 8장 완성
- Legendary 8장 수집
- 카드별 5성 달성
- 전체 컬렉션 점수 상승
- 대표 카드 설정
- 감성적인 카드 상세 화면 감상

## 7. 커피 카드 카테고리별 역할

### 7.1 에스프레소/기본 커피

초반 기본 수집군.  
커피 테마의 입문 카드들이며, 가장 먼저 도감이 채워지는 영역이다.

대표 카드:

- 아메리카노
- 에스프레소
- 도피오
- 룽고
- 리스트레토
- 카페 크레마
- 콜드브루
- 롱 블랙

UX 역할:

- 첫 수집 성취 제공
- Common/Uncommon 중심
- 카드 오픈 경험을 빠르게 학습시키는 구간

### 7.2 기본 우유 커피

부드러운 이미지의 초반~중반 수집군.

대표 카드:

- 카페 라떼
- 카푸치노
- 플랫 화이트
- 코르타도
- 카페 오레
- 밀크 에스프레소
- 오트 라떼
- 브레베 라떼

UX 역할:

- 카드 이미지 만족도 상승
- 라떼 아트, 우유층, 거품 등 시각적 차별화
- Rare 카드의 첫 목표 제공

### 7.3 시럽/달달 라떼

색감과 토핑 차이가 명확한 중반 수집군.

대표 카드:

- 바닐라 라떼
- 카라멜 라떼
- 헤이즐넛 라떼
- 허니 라떼
- 메이플 라떼
- 시나몬 라떼
- 흑당 라떼
- 솔티드 카라멜 라떼

UX 역할:

- 단맛/시럽/토핑으로 카드 차별화
- Rare 카드 수집 욕구 강화

### 7.4 모카/디저트 커피

디저트성 비주얼이 강한 중반~상위 수집군.

대표 카드:

- 카페 모카
- 화이트 모카
- 초코 라떼
- 아포가토
- 크림 모카
- 티라미수 라떼
- 코코아 에스프레소
- 디저트 콘파냐

UX 역할:

- 고급스러운 카드 이미지 확보
- Signature 카드 첫 기대감 제공
- “예쁜 카드” 만족도가 높아야 하는 구간

### 7.5 말차/티 라떼

커피와 다른 색상 톤을 제공하는 테마 변화 구간.

대표 카드:

- 말차 라떼
- 호지차 라떼
- 홍차 라떼
- 얼그레이 라떼
- 차이 라떼
- 말차 에스프레소
- 로열 밀크티
- 자스민 티 라떼

UX 역할:

- 초록/차색/꽃향 색감으로 시각 피로 완화
- 커피 카드팩 안에서도 다양성 제공

### 7.6 청량/아이스/논커피

차갑고 투명한 비주얼의 변주 구간.

대표 카드:

- 레모네이드 커피
- 스파클링 에이드
- 아이스 피치티
- 에스프레소 토닉
- 프루트 커피 피즈
- 민트 쿨러
- 자몽 콜드브루
- 블루 소다

UX 역할:

- 얼음, 탄산, 과일, 투명감으로 시각적 신선함 제공
- 여름 이벤트/광고 보상 캡슐과 궁합 좋음

### 7.7 희귀 재료 음료

고급 재료감과 풍부한 비주얼이 필요한 상위 수집군.

대표 카드:

- 희귀 원두 라떼
- 허니 크림 에스프레소
- 말차 크림 모카
- 과일 크림 라떼
- 골드 바닐라 라떼
- 희귀 카라멜 샷
- 흑임자 라떼
- 밤 크림 라떼

UX 역할:

- Signature 카드 밀도 증가
- 프리미엄 카드 이미지 확보
- 고급 캡슐의 주요 목표

### 7.8 시간대 한정 음료

시간 조건으로 열리는 이벤트성 수집군.

대표 카드:

- 모닝 미스트 라떼
- 정오의 시트러스 커피
- 이브닝 카라멜 크레마
- 나이트 벨벳 모카
- 새벽 허니 샷
- 선셋 티 라떼
- 미드나잇 토닉
- 떠돌이 블렌드

UX 역할:

- 시간대 접속 이유 제공
- 광고 보고 추가 열기와 궁합 좋음
- 매일 루틴화 가능

### 7.9 시그니처 음료

커피 테마의 대표 고급 카드군.

대표 카드:

- 하우스 시그니처
- 화이트 가든 라떼
- 벨벳 크레마
- 아틀리에 모카
- 로즈 바닐라 라떼

## 11. 카드 성장 설계

### 11.1 별 등급

카드는 최초 획득 시 1성으로 시작한다.

| 별 등급 | 조건 |
|---|---:|
| 1성 | 최초 획득 |
| 2성 | 중복 1장 |
| 3성 | 중복 3장 |
| 4성 | 중복 6장 |
| 5성 | 중복 10장 |

### 11.2 카드 점수

```text
카드 점수 = 등급 기본 점수 + 별 등급 보너스
```

### 11.3 등급 기본 점수

| 등급 | 기본 점수 |
|---|---:|
| Common | 10 |
| Uncommon | 25 |
| Rare | 60 |
| Signature | 150 |
| Legendary | 400 |

### 11.4 별 등급 보너스

| 별 등급 | 보너스 배율 |
|---|---:|
| 1성 | x1.0 |
| 2성 | x1.2 |
| 3성 | x1.5 |
| 4성 | x2.0 |
| 5성 | x3.0 |

### 11.5 컬렉션 점수

```text
컬렉션 점수 = 보유한 모든 카드의 카드 점수 합계
```

## 12. 도감 설계

### 12.1 도감의 핵심 역할

도감은 단순 목록이 아니라, 이 게임의 핵심 욕구를 만드는 화면이다.

도감은 다음을 제공해야 한다.

- 수집 진행도
- 미획득 카드의 궁금증
- 카테고리 완성 목표
- 희귀 카드 보유 자랑
- 카드 상세 감상
- 대표 카드 설정 욕구
- 다시 캡슐을 열고 싶은 동기

### 12.2 도감 화면 구성

```text
[커피 컬렉션]

전체 수집률 12 / 80
컬렉션 점수 1,240
대표 카드: 카페 라떼

카테고리 탭/리스트

에스프레소/기본 커피 4 / 8
[카드] [카드] [카드] [카드]
[잠금] [잠금] [잠금] [잠금]

기본 우유 커피 2 / 8
...
```

### 12.3 미획득 카드 표현

미획득 카드는 완전히 빈칸이 아니라, 욕구를 자극해야 한다.

표현 방식:

- 어두운 실루엣
- 등급만 표시 가능
- 이름은 `????` 또는 잠김 처리
- 카테고리 위치는 유지
- Legendary는 더 신비로운 잠금 연출

### 12.4 획득 카드 표현

획득 카드는 다음 정보를 보여준다.

- 카드 이미지
- 이름
- 등급
- 별 등급
- 보유 수량
- 신규 획득 배지
- 대표 카드 여부

### 12.5 상세 모달

상세 모달 정보:

- 큰 카드 이미지
- 카드명
- 등급
- 설명
- 카테고리
- 획득일
- 보유 수량
- 별 등급
- 다음 별까지 필요한 중복 수량
- 컬렉션 점수
- 대표 카드로 설정 버튼

## 13. 수집 욕구 강화 장치

### 13.1 카테고리 완성 보상

각 카테고리 8장 완성 시 보상.

예시:

| 카테고리 | 완성 보상 |
|---|---|
| 에스프레소/기본 커피 | 기본 커피 캡슐 3개 |
| 기본 우유 커피 | 라떼 캡슐 2개 |
| 시럽/달달 라떼 | 커피 파우더 300 |
| 모카/디저트 커피 | 디저트 캡슐 2개 |
| 말차/티 라떼 | 티 라떼 캡슐 2개 |
| 청량/아이스/논커피 | 청량 캡슐 2개 |
| 희귀 재료 음료 | 시그니처 조각 3개 |
| 시간대 한정 음료 | 시간대 캡슐 3개 |
| 시그니처 음료 | 프리미엄 캡슐 5개 |
| 전설/고급 컬렉션 음료 | 전설 카드 프레임 |

v0.1에서는 보상 UI만 기획해두고 실제 지급은 v0.2부터 구현한다.

### 13.2 세트 효과

카드 그룹을 모두 모으면 세트 효과를 준다.

예시:

```text
기본 커피 세트
- 조건: 에스프레소/기본 커피 8종 수집
- 효과: 커피 테마 컬렉션 점수 +3%

우유 커피 세트
- 조건: 기본 우유 커피 8종 수집
- 효과: 캡슐 오픈 결과 화면에서 라떼 카드 전용 배경 해금

전설의 잔 세트
- 조건: Legendary 8종 수집
- 효과: 전설 카드 프레임 해금
```

### 13.3 대표 카드

유저가 마음에 드는 카드를 대표 카드로 설정할 수 있다.

대표 카드는 다음 위치에 노출된다.

- 테마 홈
- 프로필 영역
- 테마 선택 화면
- 결과 공유 이미지
- 향후 랭킹 카드

### 13.4 오늘의 추천 카드

매일 하나의 미획득 카드를 추천한다.

예시:

```text
오늘의 추천 카드
카페 모카
모카/디저트 커피 카테고리에서 아직 발견하지 못했어요.
```

v0.2 이후 구현.

### 13.5 획득 히스토리

최근 획득 카드 10개를 보여준다.

효과:

- 유저가 성취를 다시 볼 수 있음
- 방금 뽑은 카드의 만족감을 유지
- 중복이라도 누적감이 생김

## 14. 광고 보상 설계

## 16. 애니메이션 가이드

### 16.1 핵심 원칙

애니메이션은 부드럽고 감성적이어야 한다.  
속도는 빠르지만 거칠면 안 된다.

원칙:

- 짧게
- 부드럽게
- 명확하게
- 보상 순간에만 화려하게
- 일반 UI 전환은 과하지 않게
- 모바일 저사양에서도 버벅이지 않게

### 16.2 추천 애니메이션 타이밍

| 상황 | 시간 |
|---|---:|
| 버튼 눌림 | 80~120ms |
| 카드 hover/press | 120~180ms |
| 모달 등장 | 180~260ms |
| 캡슐 흔들림 | 600~900ms |
| 카드 뒤집기 | 450~700ms |
| 결과 등장 | 300~500ms |
| Legendary 등장 | 900~1400ms |

### 16.3 캡슐 오픈 시퀀스

```text
1. 캡슐이 살짝 떠오름
2. 좌우로 미세하게 흔들림
3. 빛이 내부에서 새어 나옴
4. 탭/클릭 입력 또는 자동 개봉
5. 캡슐이 열리며 빛 파티클
6. 카드 뒷면 등장
7. 카드가 천천히 뒤집힘
8. 등급 연출
9. 카드 앞면 고정
10. 결과 버튼 등장
```

### 16.4 등급별 연출

#### Common

- 짧은 빛
- 작은 파티클
- 즉시 카드 표시

#### Uncommon

- 약한 색상 글로우
- 카드가 살짝 확대

#### Rare

- 화면 중앙 빛 번짐
- 카드 뒷면 딜레이
- 파란/금빛 포인트

#### Signature

- 배경 어두워짐
- 고급 파티클
- 별빛 회전
- 카드 등장 전 0.5초 기대감

#### Legendary

- 캡슐이 열리기 전 강한 빛
- 짧은 정적
- 화면 전체 글로우
- 카드 실루엣 먼저 공개
- 금빛 파티클 폭발
- 전용 사운드 필요

### 16.5 페이지 전환

- 테마 선택 → 테마 홈: fade + slight slide
- 홈 → 도감: slide up
- 도감 → 카드 상세: modal scale/fade
- 캡슐 결과 → 홈: fade out

### 16.6 구현 주의

- CSS transition 우선
- 필요한 경우 Framer Motion 사용
- 캡슐 오픈 연출은 컴포넌트 단위로 격리
- 애니메이션 상태는 단순 enum으로 관리

예시:

```ts
type OpenAnimationState =
  | "idle"
  | "shaking"
  | "glowing"
  | "revealing"
  | "cardFlip"
  | "result";
```

## 23. Cursor Auto 작업 원칙

### 23.1 한 번에 시키지 말 것


- 빌드 에러가 생기기 쉬움


## 29. 최종 방향

캡슐 컬렉터는 단순히 광고를 보기 위한 게임이 아니다.  
핵심은 **보상을 보고 싶고, 카드를 모으고 싶고, 도감을 채우고 싶게 만드는 수집형 감성 게임**이다.

커피 테마는 첫 번째 검증용 테마다.  
이 테마에서 다음을 증명해야 한다.

```text
캡슐을 열고 싶다.
카드가 예쁘다.
도감을 채우고 싶다.
중복도 의미 있다.
광고를 보면 추가 보상이 명확하다.
다른 테마도 나오면 모으고 싶다.
```

이 기준을 만족하면, 이후 Delta Nova, Dark RPG, 신규 게임 테마를 같은 구조에 자연스럽게 추가할 수 있다.
