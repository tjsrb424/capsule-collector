# ANIMATION GUIDE

이 문서는 `캡슐 컬렉터`의 부드럽고 감성적인 애니메이션 방향을 정리한다.  
원문 기준: `docs/CAPSULE_COLLECTOR_MASTER_DESIGN.md`

---

## 1) 핵심 원칙

- 애니메이션은 **부드럽고 감성적**이어야 한다.
- 속도는 빠르되, 거칠거나 과하면 안 된다.
- **보상 순간(캡슐 오픈/결과)만 화려하게**, 그 외 UI 전환은 절제한다.
- 모바일 저사양에서도 버벅이지 않게 설계한다.

구현 우선순위(원칙):

- 기본은 **CSS transition**으로 해결
- 필요할 때만 **Framer Motion을 제한적으로 사용**
- 캡슐 오픈 연출은 **컴포넌트 단위로 격리**
- 애니메이션 상태는 단순 enum으로 관리

예시(기획):

```ts
type OpenAnimationState =
  | "idle"
  | "shaking"
  | "glowing"
  | "revealing"
  | "cardFlip"
  | "result";
```

## 2) 추천 타이밍 가이드

| 상황 | 시간 |
|---|---:|
| 버튼 눌림 | 80~120ms |
| 카드 press | 120~180ms |
| 모달 등장 | 180~260ms |
| 캡슐 흔들림 | 600~900ms |
| 카드 뒤집기 | 450~700ms |
| 결과 등장 | 300~500ms |
| Legendary 등장 | 900~1400ms |

---

## 3) 캡슐 오픈 시퀀스(단계별)

```text
1. 캡슐이 살짝 떠오름
2. 좌우로 미세하게 흔들림
3. 빛이 내부에서 새어 나옴
4. 탭/클릭 입력 또는 자동 개봉
5. 캡슐이 열리며 빛 파티클
6. 카드 뒷면 등장
7. 카드가 천천히 뒤집힘
8. 등급 연출
9. 카드 앞면 고정
10. 결과 버튼 등장
```

의도:

- 1~3단계에서 기대감을 만들고,
- 6~9단계에서 “획득 만족”을 확정한다.

---

## 4) 등급별 연출 강도(차별화)

### Common

- 짧은 빛
- 작은 파티클
- 즉시 카드 표시

### Uncommon

- 약한 색상 글로우
- 카드가 살짝 확대

### Rare

- 화면 중앙 빛 번짐
- 카드 뒷면 딜레이(기대감)
- 파란/금빛 포인트

### Signature

- 배경 어두워짐
- 고급 파티클
- 별빛 회전
- 카드 등장 전 약 0.5초 기대감

### Legendary

- 캡슐이 열리기 전 강한 빛
- 짧은 정적
- 화면 전체 글로우
- 카드 실루엣 먼저 공개 → 공개(단계감)
- 금빛 파티클 폭발
- 전용 사운드 필요(향후)

## 5) 페이지 전환 / 모달 / 인터랙션

### 5.1 페이지 전환(기획)

- 테마 선택 → 테마 홈: fade + slight slide
- 홈 → 도감: slide up
- 도감 → 카드 상세: modal scale/fade
- 캡슐 결과 → 홈: fade out

### 5.2 모달 등장(원칙)

- 등장/사라짐은 200ms 전후의 scale + fade로 “부드럽게”
- 배경 dim은 너무 어둡지 않게(감성 유지)

### 5.3 카드 뒤집기(원칙)

- 뒤집기는 450~700ms 범위에서 “느리지만 답답하지 않게”
- Rare 이상은 뒤집기 시작 전 150~300ms 정도의 숨 고르기(기대감) 가능

### 5.4 버튼 눌림/피드백

- press 상태는 80~120ms로 빠르게 반응
- 가벼운 scale-down + shadow 변화로 촉감 제공

---

## 6) 모바일 성능 가이드(제약)

- 긴 지속 애니메이션/대형 blur 남발 금지
- 레이아웃 변경(reflow)을 많이 유발하는 애니메이션 최소화
- transform/opacity 중심으로 구성
- 파티클/글로우는 “짧고 강하게” (보상 순간에만)
