# Sprint 4: 수집 욕구 UI 고도화

## 1. 목표

도감과 카드 UI를 고도화해서 “계속 모으고 싶다”는 감각을 강화한다.

이번 스프린트는 기능 추가보다 **수집 욕구를 자극하는 정보 배치와 시각 표현**에 집중한다.

---

## 2. 구현 범위

### 2.1 미획득 실루엣 개선

미획득 카드는 단순 빈칸이 아니라 기대감을 줘야 한다.

표현:

```text
실루엣
잠금 아이콘
카테고리 위치 유지
등급 힌트 optional
???? 표시
```

Legendary 미획득 카드는 더 특별한 잠금 상태로 보일 수 있다.

### 2.2 신규 배지

방금 획득한 카드는 도감에서 `NEW` 배지를 보여준다.

요구사항:

- 최근 획득 카드 기준
- 너무 오래 유지되지 않게 향후 제거 가능
- v0.1에서는 local state 또는 저장된 timestamp 기반으로 처리 가능

### 2.3 별 등급 표시

카드에 별 등급을 표시한다.

```text
★
★★
★★★
★★★★
★★★★★
```

미획득 카드는 별 표시 없음.

### 2.4 최근 획득 카드

커피 테마 홈에 최근 획득 카드 5장을 표시한다.

효과:

- 방금 얻은 카드의 만족감 유지
- 수집 진행이 눈에 보임

### 2.5 대표 카드 설정

카드 상세 모달에서 대표 카드로 설정할 수 있게 한다.

저장 위치:

```text
UserThemeState.representativeCardId
```

없으면 타입 확장한다.

표시 위치:

- 커피 테마 홈
- 메인 테마 카드

### 2.6 카테고리 완성 UI 준비

실제 보상 지급은 나중에 하더라도, 카테고리 완성 상태를 표시한다.

```text
완성!
8 / 8
보상 준비 중
```

## 3. 금지 범위

이번 스프린트에서는 하지 않는다.

```text
카테고리 완성 보상 실제 지급
상점
랭킹
실제 광고 SDK
고급 캡슐 종류 추가
```

---

## 4. 완료 기준

- 미획득 카드가 매력적인 잠금 상태로 보인다.
- 신규 배지가 표시된다.
- 별 등급이 표시된다.
- 최근 획득 카드가 홈에 표시된다.
- 대표 카드 설정이 가능하다.
- 카테고리 완성 상태가 시각적으로 표현된다.

---

## 5. 테스트 방법

```bash
npm run dev
```

확인:

```text
1. 캡슐을 열어 카드 획득
2. 도감에서 NEW 배지 확인
3. 중복 획득 후 별 등급 변화 확인
4. 카드 상세에서 대표 카드 설정
5. 홈에서 대표 카드 표시 확인
6. 최근 획득 카드 표시 확인
```
