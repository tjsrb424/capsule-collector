# Sprint 0-B: 커피 테마 홈 UX 정리

## 1. 목표

Sprint 0-A에서 구축한 수집형 엔진을 바탕으로, **테마 선택 화면과 커피 테마 홈 화면의 UX를 정리**한다.

이번 스프린트의 핵심은 “캡슐 컬렉터가 어떤 게임인지 첫 화면에서 이해되고, 커피 테마에 들어가고 싶게 만드는 것”이다.

---

## 2. 구현 범위

### 2.1 메인 화면

메인 화면은 캡슐 컬렉터의 상위 허브다.

표시 요소:

```text
캡슐 컬렉터
여러 게임 테마의 카드를 모아보세요.

[커피 컬렉션]
수집률 n / total
컬렉션 점수
대표 카드 영역 또는 placeholder
입장하기
```

나중에 추가될 테마는 Coming Soon으로 표시할 수 있다.

```text
[델타노바 컬렉션] Coming Soon
[다크 RPG 컬렉션] Coming Soon
```

### 2.2 커피 테마 홈

커피 테마 홈은 커피 수집의 중심 화면이다.

필수 정보:

```text
커피 컬렉션
수집률
컬렉션 점수
커피 파우더
시그니처 조각
전설 원두 조각
오늘 광고 캡슐 사용 횟수
```

CTA:

```text
[기본 커피 캡슐 열기]
[광고 보고 보너스 캡슐]
[도감 보기]
```

### 2.3 대표 카드 영역

아직 대표 카드 설정 기능이 없어도 placeholder로 준비한다.

상태:

```text
대표 카드 없음
첫 카드를 획득하면 여기에 표시됩니다.
```

보유 카드가 있으면 최근 획득 카드 또는 첫 번째 보유 카드를 임시 대표처럼 보여줄 수 있다.

### 2.4 최근 획득 카드 Preview

v0.1에서는 완전한 히스토리 저장이 없어도 된다.  
현재 보유 카드 중 최근 획득일 기준으로 간단히 3~5장을 보여줄 수 있다.

### 2.5 라우팅 정리

가능하면 경로를 명확히 한다.

```text
/                  메인 테마 선택
/theme/coffee      커피 테마 홈
/collection/coffee 커피 도감, Sprint 2에서 본격 구현
```

아직 `/collection/coffee`가 없으면 버튼은 disabled 또는 Coming Soon 처리할 수 있다.

---

## 3. UI 방향

- 모바일 세로 화면 우선
- 따뜻한 커피 톤
- 깔끔하고 모던한 카드 UI
- 과한 장식보다 수집률과 캡슐 CTA가 잘 보여야 함
- 버튼은 크고 명확해야 함
- 홈 화면에서 “수집형 게임” 느낌이 나야 함

권장 색감:

```text
Cream
Latte
Caramel
Coffee Brown
Soft Gold
```

---

## 4. 금지 범위

이번 스프린트에서는 하지 않는다.

```text
도감 전체 구현
80장 데이터 입력
실제 광고 SDK
고급 캡슐 오픈 애니메이션
카드 상세 모달
대표 카드 설정 저장 기능
복잡한 상점
```

---

## 5. 완료 기준

- 메인 화면에서 커피 테마가 명확히 보인다.
- 커피 테마 홈으로 진입 가능하다.
- 커피 테마 홈에서 수집률, 점수, 재화가 보인다.
- 기본 캡슐 열기 CTA가 명확하다.
- 광고 보너스 캡슐 CTA가 mock임을 알 수 있다.
- 전체 UI가 모바일 세로에서 깨지지 않는다.

---

## 6. 테스트 방법

```bash
npm run dev
```

확인:

```text
1. / 접속
2. 커피 컬렉션 카드 확인
3. 커피 테마 홈 진입
4. 수집률/점수/재화 표시 확인
5. 기본 캡슐 열기 동작 확인
6. 광고 보너스 캡슐 버튼 표시 확인
7. 모바일 폭으로 화면 확인
```
