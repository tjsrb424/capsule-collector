# Sprint 6: 광고 보상 mock 고도화

## 1. 목표

실제 광고 SDK 없이도 광고 보상 흐름을 검증할 수 있게 한다.

이번 스프린트는 “광고를 보면 무엇을 얻는지”가 명확해야 한다.  
실제 광고 SDK는 붙이지 않는다.

---

## 2. 구현 범위

### 2.1 mock 광고 서비스

파일:

```text
lib/ads/mockRewardedAd.ts
```

예상 함수:

```ts
watchMockRewardedAd(rewardType)
```

동작:

```text
1. 광고 로딩 상태
2. 1초 대기
3. 성공 반환
4. 보상 지급
```

### 2.2 광고 보고 보너스 캡슐

커피 테마 홈에서 사용.

```text
[광고 보고 보너스 캡슐]
```

동작:

```text
클릭
→ mock 광고 재생
→ coffee_ad_bonus_capsule 1회 오픈
→ 결과 표시
→ dailyAdOpenCount 증가
```

### 2.3 중복 보상 2배

결과가 중복일 때만 표시.

```text
[광고 보고 보상 2배]
```

동작:

```text
클릭
→ mock 광고 재생
→ 이번 중복 보상 추가 지급
→ 버튼 비활성화
→ dailyAdRewardDoubleCount 증가
```

### 2.4 일일 제한

기본값:

```text
광고 보너스 캡슐: 하루 10회
중복 보상 2배: 하루 10회
```

v0.1에서는 실제 날짜 리셋이 단순해도 된다.  
가능하면 `lastAdDate`를 저장해 날짜가 바뀌면 초기화한다.

---

## 3. UI 요구

광고 버튼은 보상이 명확해야 한다.

좋은 문구:

```text
광고 보고 보너스 캡슐 열기
광고 보고 커피 파우더 2배
오늘 남은 광고 보상 7/10
```

광고 로딩 중:

```text
광고 확인 중...
```

성공 후:

```text
보상 지급 완료
```

---

## 4. 금지 범위

이번 스프린트에서는 하지 않는다.

```text
실제 광고 SDK
AdMob 연동
AdSense/H5 광고 연동
광고 네트워크 선택
서버 검증
결제
```

---

## 5. 완료 기준

- 광고 보너스 캡슐 mock이 동작한다.
- 중복 보상 2배 mock이 동작한다.
- 일일 제한 표시가 있다.
- 광고 로딩/성공 상태가 명확하다.
- 실제 광고 SDK가 추가되지 않았다.

---

## 6. 테스트 방법

```bash
npm run dev
```

확인:

```text
1. 커피 테마 홈 접속
2. 광고 보고 보너스 캡슐 클릭
3. mock 광고 대기 확인
4. 보너스 캡슐 결과 확인
5. 중복 카드 발생 시 보상 2배 버튼 확인
6. 보상 2배 클릭 후 재화 증가 확인
7. 일일 카운트 감소 또는 증가 확인
```
