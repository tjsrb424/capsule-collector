# Sprint 2: 커피 도감 화면 구현

## 1. 목표

커피 테마의 카드들을 **도감 화면에서 카테고리별로 확인**할 수 있게 한다.

도감은 이 게임의 핵심 장기 동기다.  
단순 리스트가 아니라 “빈칸을 채우고 싶다”는 감각을 줘야 한다.

---

## 2. 구현 범위

### 2.1 경로

```text
/collection/coffee
```

### 2.2 도감 상단 요약

표시:

```text
커피 컬렉션
전체 수집률 n / total
컬렉션 점수
보유 카드 수
전체 카드 수
```

### 2.3 카테고리 그룹

카드를 `categoryId` 기준으로 그룹화한다.

표시:

```text
에스프레소/기본 커피 4 / 8
[카드] [카드] [잠금] [잠금]
```

### 2.4 획득 카드 표현

획득 카드:

```text
카드명
등급
별 등급
보유 수량
```

### 2.5 미획득 카드 표현

미획득 카드:

```text
???? 또는 잠금
실루엣/어두운 placeholder
등급 힌트는 선택적으로 표시
```

### 2.6 카드 상세 모달

카드를 누르면 상세 모달을 연다.

획득 카드 상세:

```text
카드명
등급
설명
카테고리
보유 수량
별 등급
컬렉션 점수
다음 별까지 필요한 수량
```

미획득 카드 상세:

```text
아직 발견하지 못한 카드입니다.
이 카테고리의 캡슐을 열어보세요.
```

## 4. 금지 범위

이번 스프린트에서는 하지 않는다.

```text
80장 데이터 입력
대표 카드 설정
최근 획득 히스토리
카테고리 완성 보상 지급
고급 애니메이션
실제 이미지 대량 적용
```

---

## 5. 완료 기준

- `/collection/coffee` 접속 가능
- 카드가 카테고리별로 표시됨
- 획득/미획득 상태가 구분됨
- 카드 상세 모달이 동작함
- 수집률이 정확히 표시됨
- 모바일 세로 화면에서 보기 좋음

---

## 6. 테스트 방법

```bash
npm run dev
```

확인:

```text
1. 캡슐 몇 번 열기
2. /collection/coffee 이동
3. 획득 카드가 앞면으로 보이는지 확인
4. 미획득 카드가 잠금으로 보이는지 확인
5. 카드 클릭 시 상세 모달 확인
6. 수집률 갱신 확인
```
