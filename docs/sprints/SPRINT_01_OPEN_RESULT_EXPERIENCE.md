# Sprint 1: 캡슐 오픈 결과 경험 개선

## 1. 목표

캡슐을 열었을 때 결과가 단순 텍스트가 아니라 **보상감을 주는 수집형 카드 획득 경험**으로 느껴지게 개선한다.

이번 스프린트는 결과 모달과 오픈 경험을 다듬는다.  
복잡한 애니메이션은 Sprint 5에서 진행하고, 이번에는 정보 구조와 기본 연출에 집중한다.

---

## 2. 구현 범위

### 2.1 결과 모달 개선

결과 모달에 다음 정보를 표시한다.

```text
카드명
등급
카테고리
설명
신규/중복 여부
중복 보상
보유 수량
별 등급
다시 열기 버튼
닫기 버튼
```

### 2.2 신규 카드 체감

신규 카드일 때는 다음이 명확해야 한다.

```text
NEW!
새 카드 발견
도감에 등록되었습니다.
```

시각 효과:

- NEW 배지
- 카드 강조 테두리
- 밝은 배경 글로우
- 카드가 살짝 커지는 등장 효과

### 2.3 중복 카드 체감

중복 카드일 때는 실망감을 줄이고 성장 재화로 전환되는 느낌을 준다.

표시:

```text
중복 카드
커피 파우더 +5
보유 수량 3
다음 별까지 1장
```

### 2.4 등급별 기본 색상

등급별 최소 색상 체계를 적용한다.

| rarity | 방향 |
|---|---|
| common | 크림/브라운 |
| uncommon | 브론즈/그린 |
| rare | 블루/골드 |
| signature | 퍼플/골드 |
| legendary | 골드/화이트 |

### 2.5 다시 열기 UX

결과 모달에서 바로 다시 열 수 있어야 한다.

버튼:

```text
[한 번 더 열기]
[닫기]
```

중복일 때:

```text
[광고 보고 보상 2배] // Sprint 6에서 실제 mock 고도화, 이번에는 자리만 가능
[한 번 더 열기]
```

## 3. 금지 범위

이번 스프린트에서는 하지 않는다.

```text
완전한 캡슐 오픈 시퀀스
카드 뒤집기 고급 애니메이션
실제 광고 SDK
도감 상세 화면
80장 데이터 입력
복잡한 사운드
```

---

## 4. 완료 기준

- 결과 모달에서 신규/중복이 명확히 구분된다.
- 중복 보상이 시각적으로 표시된다.
- 다시 열기 버튼으로 연속 오픈이 가능하다.
- 등급별 기본 스타일이 적용된다.
- 모바일 화면에서 모달이 보기 좋다.

---

## 5. 테스트 방법

```bash
npm run dev
```

확인:

```text
1. 기본 캡슐 열기
2. 신규 카드 결과 확인
3. 여러 번 열어 중복 카드 확인
4. 중복 보상 표시 확인
5. 한 번 더 열기 버튼 확인
6. 등급별 스타일 확인
```
