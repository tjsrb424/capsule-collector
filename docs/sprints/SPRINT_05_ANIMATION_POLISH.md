# Sprint 5: 애니메이션 고도화

## 1. 목표

캡슐 오픈과 카드 획득 순간을 **부드럽고 감성적인 보상 경험**으로 개선한다.

이번 스프린트는 게임의 감정 품질을 높이는 단계다.  
단, 모바일 성능을 해치지 않도록 과한 애니메이션은 피한다.

---

## 2. 구현 범위

### 2.1 캡슐 오픈 시퀀스

상태 기반으로 구현한다.

```ts
type OpenAnimationState =
  | "idle"
  | "shaking"
  | "glowing"
  | "revealing"
  | "cardFlip"
  | "result";
```

시퀀스:

```text
1. 캡슐 대기
2. 캡슐 흔들림
3. 내부 빛
4. 캡슐 열림
5. 카드 뒷면 등장
6. 카드 뒤집기
7. 등급 연출
8. 결과 모달/결과 카드 표시
```

### 2.2 카드 뒤집기

카드가 뒷면에서 앞면으로 뒤집히는 기본 연출.

요구사항:

- 450~700ms
- transform 기반
- 모바일에서 버벅이지 않게
- 결과 데이터는 뒤집기 중간 이후 공개

### 2.3 등급별 연출 강도

| rarity | 연출 |
|---|---|
| common | 짧은 빛, 작은 파티클 |
| uncommon | 약한 글로우 |
| rare | 중앙 빛 번짐, 살짝 긴 딜레이 |
| signature | 배경 어두워짐, 별빛 파티클 |
| legendary | 강한 금빛 글로우, 짧은 정적, 전용 강조 |

### 2.4 모달 전환

- fade + scale
- 180~260ms
- 너무 튀지 않게

### 2.5 페이지 전환

필요한 경우에만 적용.

```text
테마 선택 → 테마 홈: fade + slide
홈 → 도감: slide up
도감 → 상세: modal
```

---

## 3. 구현 방식

권장:

- CSS transition 우선
- 필요한 경우 Framer Motion 사용
- 애니메이션 컴포넌트를 캡슐 오픈 영역에 격리
- 상태 전환은 단순하게 유지

주의:

- requestAnimationFrame 과다 사용 금지
- 파티클 DOM 과다 생성 금지
- 무한 애니메이션 남발 금지
- 저사양 모바일 고려

---

## 4. 금지 범위

이번 스프린트에서는 하지 않는다.

```text
사운드 추가
복잡한 3D 연출
무거운 캔버스 파티클
실제 광고 SDK
새로운 게임 시스템 추가
```

---

## 5. 완료 기준

- 캡슐 오픈 시퀀스가 단계적으로 보인다.
- 카드 뒤집기 연출이 있다.
- 등급별 결과 체감 차이가 있다.
- 모달/버튼 전환이 부드럽다.
- 모바일 화면에서 과하게 버벅이지 않는다.

---

## 6. 테스트 방법

```bash
npm run dev
```

확인:

```text
1. 기본 캡슐 열기
2. 캡슐 흔들림 확인
3. 카드 뒤집기 확인
4. 등급별 결과 연출 확인
5. 여러 번 반복해도 UI가 깨지지 않는지 확인
6. 모바일 폭에서 확인
```
